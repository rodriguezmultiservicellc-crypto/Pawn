  KEY DECISIONS LOCKED THIS SESSION (Session 37):
    - CUSTOMER PICKER ARCHITECTURE — server
      typeahead with debounced ilike, no client-side
      filter. Top 20 results. Same picker reused
      across 5 forms via prop config (enableDlScan,
      initialCustomerId, onChange, ref). One server
      action (searchCustomersForPicker). Picker
      manages its own selection state internally;
      forms read via the hidden input or via the
      onChange callback.
    - CUSTOMER NAV: /customers list → DASHBOARD
      first. Edit page is reached via the Edit pill
      on the dashboard. Inverts the prior flow.
      Operator confirmed via direct request.
    - MARKET DATA = ANONYMIZED + SUPERADMIN-ONLY.
      source_tenant_id stored for audit but NEVER
      returned downstream. RLS policy = read only
      when profiles.role='superadmin'. No INSERT/
      UPDATE/DELETE policies — only service-role +
      SECURITY DEFINER triggers + the search RPC
      can write or query. Per-tenant opt-out via
      tenants.share_market_data (default TRUE).
    - PGVECTOR + OPENAI EMBEDDINGS, NOT RECHARTS.
      The lookup uses pgvector cosine similarity
      via market_data_search_by_embedding RPC.
      Embedding model = text-embedding-3-small
      (1536 dims). No client-side chart library
      added — admin UI uses pure stat-tile cards.
    - FORWARD-ONLY COLLECTION. Triggers fire on
      new transactions starting after migration
      0036. Historical backfill is Phase C
      (deferred until query patterns validated).
    - PAWN CATEGORIES = TWO-LEVEL HIERARCHY,
      OPERATOR-EDITABLE. Top-levels have
      parent_id IS NULL; subs point to a parent
      via self-FK with ON DELETE CASCADE. Unique
      index scoped by COALESCE(parent_id, ZERO_
      UUID) so a sub of one top can share a slug
      with a sub of another top. Picker drilled-
      in state local to component; onPick fires
      when (a) sub is clicked OR (b) top with no
      subs is clicked. Voice intake auto-picks
      'general' (no subs) so the wizard never
      blocks voice flow.
    - HAS_FIREARMS GATE = OWNER/CHAIN_ADMIN ONLY.
      Manager can edit category list but cannot
      flip has_firearms (FFL is a compliance
      boundary). Default FALSE so no shop
      accidentally thinks it can pawn firearms
      through this app — currently the wizard
      tile is the ONLY work product; there's no
      4473/NICS workflow yet (Phase 3, deferred).
    - PHOSPHOR HAS NO GUN ICON. Used Crosshair
      as the closest semantic fit for the
      Firearms tile + sub-tiles. Operator-
      editable in /settings/pawn-categories
      icon picker.
    - REACT-HOOKS/STATIC-COMPONENTS LINT WORKAROUND
      = createElement(). The CategoryIcon wrapper
      uses createElement(iconFor(name), {...})
      instead of `const Icon = iconFor(...);
      <Icon ... />` because the linter can't see
      that the lookup is referentially stable.
      Pattern documented for future icon-by-name
      wrappers.
    - TENANTS USES is_active NOT deleted_at FOR
      SOFT-SUSPEND. The 0037 migration originally
      filtered by deleted_at IS NULL in the
      backfill loop and failed. Fixed in 56d53ea.
      Worth a CLAUDE.md gotcha entry for future
      migration authors. (See KNOWN GOTCHAS list
      below.)
    - BOUNDARY-CAST PATTERN FOR NOT-YET-TYPED
      TABLES. New tables/RPCs created by 0036/
      0037/0038 don't appear in generated database
      types until `npm run db:types` runs post-
      migration. Code uses targeted (admin.from
      as any)('table_name') + as unknown as T
      casts at the call boundary. Pattern: same
      file as the original cast comment, easy to
      grep `boundary cast` if/when types are
      regenerated and the casts can be removed.
