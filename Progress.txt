  KEY DECISIONS LOCKED THIS SESSION:
    - DUAL SKU FIELDS. inventory_items.sku TEXT ('INV-000001') +
      sku_number BIGINT (raw integer for sortable indexes). Trigger
      assigns both when sku is null on insert. User-provided SKUs get
      sku_number=0 (live outside the auto-numbered range).
    - PAWN FIELDS GATE BY has_pawn IN UI ONLY. Schema columns exist
      regardless — keeps customer rows portable if a shop later toggles
      has_pawn on. UI hides the section on jewelry-only / repair-only
      shops so the form stays focused.
    - AUDIT_LOG INSERTS THROUGH ADMIN CLIENT. The table has a SELECT
      policy for staff but no INSERT policy, so RLS would block client-
      scoped inserts. Helper uses admin (service-role) client and
      swallows errors — audit-log writes must NEVER break the user
      action. Write-once trigger from 0001 means rows can never be
      mutated post-insert.
    - JSON-ROUNDTRIP NARROWING for audit log changes. Supabase types
      JSONB columns as a strict Json union; Record<string, unknown> is
      too loose. JSON.parse(JSON.stringify(x)) cast satisfies the type
      without runtime cost — values are already JSON-serializable.
    - CUSTOMER PHOTO STORED AS PATH IN customers.photo_url. Column
      name is legacy (it was 'photo_url' in CLAUDE.md spec); we store
      the storage path and always go through getSignedUrl on read.
      Lives in customer-documents bucket under '<tenant>/<customer>/
      photo/<uuid>.<ext>'.
    - PHOTO PRIMARY DEMOTE-ON-DELETE. Deleting a primary photo auto-
      promotes the next-positioned photo. BEFORE INSERT/UPDATE trigger
      on inventory_item_photos.is_primary already handles the demote-
      siblings case at write time.
    - SHARED FORM-FIELDS COMPONENTS. CustomerFormFields and
      InventoryFormFields are imported by both the create and edit
      pages. Edit pages pass isEdit=true to enable read-only SKU and
      reveal sale_price.
    - DASHBOARD COUNTS VIA HEAD=TRUE. Each stat card runs a
      .select('id', { count: 'exact', head: true }) — the row payload
      is empty, only the count comes back. Six queries run in parallel
      via Promise.all at /dashboard.
