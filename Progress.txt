  KEY DECISIONS LOCKED THIS SESSION:
    - PAWN SAVE BUG was the same Session 8 root cause exposed in a
      new surface — z.enum().optional().nullable() rejects "" but
      the metal_type <select> "—" placeholder always sends "". The
      Session 8 fix only patched customers + inventory; loan.ts
      collateralItemSchema kept the bare enum form. Fix mirrors
      inventory.metal — preprocess "" → null BEFORE the enum runs.
      Lesson: any time Session 8's preprocess shield is missing on
      an enum field whose <select> has a placeholder option, that
      form will fail every save where the user doesn't pick a value.
    - ADMIN MENU ACCESS — staff topbar now shows an "Admin · Console"
      chip in the top-right gated on ctx.globalRole === 'superadmin'.
      Reciprocal "Staff →" link in the admin topbar. Proxy already
      gated /admin/* on superadmin — the UI just had no entry point.
    - SECURITY DEFINER ON next_tenant_counter ROOT CAUSE: 0003
      created the counter helper as plain plpgsql. Triggers on
      inventory_items / loans / repair_tickets / sales / etc. all
      call it from BEFORE INSERT, running in the user's transaction.
      INSERT/UPDATE on tenant_counters fired RLS as the user, but
      the table only has a SELECT policy by design ("app-only writes
      (RPC)" — comment in 0003 line 537). Non-superadmin staff
      therefore hit "new row violates row-level security policy for
      table tenant_counters" on every per-tenant counter advance.
      Fix is the documented design: SECURITY DEFINER on the function
      so it runs as postgres and bypasses RLS, with locked
      search_path per Postgres SECURITY DEFINER best practice.
      0017 keeps the body identical to 0003 — only the security
      attributes change.
    - SECURITY REVIEW PART A SHIPPED, PART B DEFERRED. External
      review surfaced 5 high-severity findings. Each was verified

    - PORTAL CROSS-TENANT FIX (#3) — resolvePortalCustomer in

  KNOWN GOTCHAS TO AVOID FROM DAY 1 (carried + new):
    - Next 16 file is `proxy.ts`, exported function named `proxy`.
    - Next 16 'use server' modules can ONLY export async functions.
    - Phosphor icons crash server components — only import in 'use
      client' files.
    - Phosphor doesn't have FlaskConical — that's Lucide. Use Flask.
    - useSearchParams() must be wrapped in <Suspense> for static gen.
    - <form action={fn}> never has explicit encType.
    - Storage uploads from server actions use admin (service-role)
      client.
    - `npm run db:types` via the safe wrapper.
    - Service role key NEVER goes in NEXT_PUBLIC_*.
    - DATABASE_URL via Supavisor pooler.
    - GitHub commits MUST be authored by the identity Vercel has
      linked.
    - git worktree cleanup uses `-f -f` (DOUBLE-force).
    - Auth callback needs both PKCE AND implicit-flow handling.
    - Portal invites: don't use Supabase built-in SMTP.
    - profiles.role is GLOBAL only ('superadmin' | NULL).
    - profiles PK column is `id` (FK to auth.users.id), NOT
      `user_id`. RLS predicates that gate on profiles.role: use
      `(SELECT role FROM profiles WHERE id = auth.uid()) =
      'superadmin'`.
    - user_tenants HAS user_id (separate table; no FK to profiles).
    - audit_log has SELECT policy only; inserts must use admin
      client.
    - audit_log.user_id IS NULLABLE — webhook events / cron jobs
      pass null. logAudit() signature accepts `string | null`.
    - CHECK-constrained TEXT columns come back as plain string from
      db:types — narrow at the page boundary.
    - Supabase JSONB columns need JSON-roundtrip narrowing.
    - Supabase update(patch) types reject Record<string, unknown>
      — type the patch as the table's Update alias instead.
    - First uploaded inventory photo auto-promotes to primary.
    - Storage path layout: tenant_id MUST be folder[0] for RLS.
    - Pawn ticket print locks core fields.
    - Forfeiture / abandonment inventory items start
      status='available'.
    - Cross-chain transfers blocked at trigger.
    - Use useMemo for derived state, not useEffect+setState.
    - Date.now() / random / location inside useState() trip the
      react-hooks/purity rule. Wrap in lazy initializer.
    - lib/audit.ts pulls in the admin client — never import from
      a client component. Use lib/audit-format.ts for label helpers.
    - layaway_payments.amount allows NEGATIVE values. CHECK is
      `<> 0`.
    - Stripe Connect: every per-tenant call needs Stripe-Account
      header. SaaS-platform calls (lib/stripe/saas.ts) must NOT set
      that header.
    - @fontsource fonts in @react-pdf/renderer must be loaded via
      fs.readFileSync + base64 dataURL — not require()/import.
    - Setting state inside useEffect on mount triggers
      react-hooks/set-state-in-effect. Use lazy state initializers
      with typeof window guards instead.
    - Debounced search MUST live in onChange handler, not useEffect.
    - 3-way merge of additive top-level objects: ALWAYS UNION.
    - When re-dispatching an agent after a crash, scrub the primary
      tree of leftover untracked files FIRST.
    - Validation Zod helpers — preprocess MUST handle empty-string-
      after-trim BEFORE the inner schema.
    - NUMERIC(18,4) columns are typed as `number` in generated DB
      types. Don't .toFixed() values before passing to Insert types
      — use the r4()-rounded number directly.
    - lib modules with 'server-only' cannot be imported by client
      components. Split: pure types/helpers in a .ts WITHOUT
      'server-only'; server-DB-touching code in a separate file
      WITH it.
    - .toFixed(4) is the wrong safety net. r4() already 4dp-rounds;
      .toFixed produces strings that break type checking against
      generated NUMERIC column types.
    - SESSION 8: <form action={fn}> in React 19 auto-resets the form
      after submission. Pattern when you want to preserve typed
      values on validation error: action returns { values: echoMap,
      fieldErrors }; client useEffect on state bumps a formGen key;
      form remounts with echoed values as new defaultValue.
    - SESSION 8: z.enum().optional().nullable() rejects "" (empty
      string). The <select> "—" placeholder option always sends "".
      Wrap the enum in a preprocess that maps "" → null FIRST.
      AUDIT EVERY enum field that backs a <select> with a
      placeholder. Session 9 found loan.ts collateralItemSchema
      had been missed in the original Session 8 sweep.
    - SESSION 8: When building a Select component, always include
      an `error` prop + red-border treatment from the start. A
      validation failure on a field with no visible highlight is
      worse than a hard crash.
    - SESSION 8: Stripe REST pattern for both Connect (per-tenant)
      and SaaS (platform) — raw fetch + URLSearchParams. Webhook
      verify uses Web Crypto HMAC-SHA256 (Edge-safe).
    - SESSION 8: Plan-tier feature gates via lib/saas/gates.ts.
      Don't gate on subscription status (past_due/unpaid) — that's
      dunning's job. Status-based degradation belongs in the webhook
      handler, not the gate.
    - SESSION 8: Parallel-agent worktree gotcha — agents that Write
      before realizing they're in a worktree leak files to main's
      filesystem. Recovery: `git show main:<path> > <main-path>`
      to restore main's content + rm to delete the agent's strays.
      Lesson: agents should `pwd` before first Write to confirm
      they're in the worktree path.
    - SESSION 8: compliance_log.source_id is polymorphic — for
      pawn_intake it points to loans.id; for buy_outright it points
      to the FIRST inventory_item.id (items_snapshot lists the
      rest). source_table='inventory_items' in the buy case.
    - SESSION 9: any postgres function that mutates a tenant-scoped
      table from inside a trigger context MUST be marked
      SECURITY DEFINER with locked search_path. The tenant_counters
      design intent ("app-only writes (RPC)") only works if the
      function bypasses RLS via DEFINER privileges. Pattern:
      LANGUAGE plpgsql SECURITY DEFINER SET search_path = public,
      pg_temp + REVOKE PUBLIC + GRANT EXECUTE TO authenticated.
    - SESSION 9: x-vercel-cron header is NOT a security check.
      Vercel Cron's actual auth is the Authorization: Bearer header
      it sets when CRON_SECRET is configured at the project level.
      Cron routes must require Authorization-only and reject
      anything else. CRON_SECRET is now a required env var.
    - SESSION 9: Portal customer resolution must verify an active
      'client' membership at customer.tenant_id specifically, not
      just trust the active-tenant cookie. The customers.auth_user
      _id linkage maps a user to ONE customer row, but the user's
      user_tenants membership at that customer's tenant must be
      actively confirmed — otherwise stale memberships from prior
      shops can leak.
    - SESSION 9: Report-style API endpoints (any /api/reports/*)
      must enforce staff role at the endpoint level. The proxy
      doesn't run on /api/* and page-level redirects don't fire
      when a route is hit programmatically. Use a STAFF_ROLES set
      check inside the API guard.
