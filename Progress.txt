      tokens (`reportColors`) intentionally
      stay separate. Paper printing has
      different contrast budgets than monitor
      rendering; charcoal-on-white reads
      better at 600dpi than navy.

      Replaced the dusty saturation-matched
      success #2D7A4E / warning #B7791F with
      Tailwind's emerald-500 / amber-500. The
      "saturation match to error red" rationale
      doesn't apply once Rausch leaves. Same
      Tailwind class names (text-success /
      bg-warning / etc.) — only the hex shifted,
      no template edits needed for those.
    - ACTION COLORS NEVER OVERLAP STATUS
      COLORS. Gold = primary action. Blue =
      link / focus. Success / warning / danger
      / info = strictly status. A "click here"
      moment never uses success green; a
      "completed" moment never uses gold.

      The Luna Azul reference shows gold + navy
