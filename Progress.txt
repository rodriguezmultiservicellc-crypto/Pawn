  KEY DECISIONS LOCKED THIS SESSION:
    - AUDIT-LOG NOTES SCAN CAPTURE FLAG, NOT RAW PAYLOAD.
      audit_log is staff-readable and the parsed DL
      fields (name, license number, expiry) already
      land in the customer row. Logging the raw AAMVA
      payload would duplicate PII into the audit
      viewer with no recovery benefit (the customer row
      itself has the parsed data + the raw payload from
      patches/0025). Audit records `dl_scan_captured:
      boolean` instead.
    - EXPIRED-LICENSE GUARD IS UI-ONLY, NOT BLOCKING.
      Pawn-shop convention varies — some shops refuse
      expired IDs entirely, others accept with
      documentation (military deployment, recent
      renewal in mail, etc.). Form shows a warning
      banner so it's a deliberate operator decision,
      but doesn't reject submit. If a tenant ever
      wants a hard block, that's a tenant-setting
      conversation, not a default.
    - WEBCAM REAR-CAM PREFERRED VIA facingMode:
      'environment'. Tablets / phones default to the
      back camera (the right side for ID capture);
      desktops fall through to the only camera they
      have. JPEG @ 92% at native video resolution
      (typically 1080p+) keeps front-of-license
      sharpness past the FL 2-year retention window.
    - WATCH→COLLATERAL USES IMPERATIVE HANDLE, NOT
      LIFTED STATE. CollateralItemsList row state
      includes File objects + preview URLs +
      keystroke-by-keystroke text inputs; lifting it
      would re-render the form on every keystroke.
      forwardRef + useImperativeHandle exposes a
      narrow surface (addWatchRow only) — the parent
      pushes a populated row in but doesn't own the
      list state. Same trade-off lib/repair/workflow's
      pure helpers + state machine make: keep
      side-effects narrow.
    - PRISTINE-SEED REPLACE-IN-PLACE FOR FIRST WATCH.
      newRow() seeds the list with a default empty row
      so the form has something to render. When the
      operator picks the FIRST watch via typeahead,
      addWatchRow detects the seed is still pristine
      (description empty + category 'other' + no photo
      + est_value '0') and replaces it in place. Saves
      a click to delete the placeholder. Subsequent
      adds append.
    - SPLIT-WRITE PATTERN EXTENDS TO INSERT. Session 17
      established the split-write for UPDATE on
      untyped columns (agency_store_id). Session 18
      extends to INSERT for dl_raw_payload — single
      `as never` cast on the whole insertPayload const
      since INSERT has no equivalent multi-statement
      version. After db:types runs, the cast becomes
      a one-line removal.
