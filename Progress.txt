  KEY DECISIONS LOCKED THIS SESSION:
    - SAVE BUG ROOT CAUSE was TWO BUGS COMPOUNDING. (a) React 19
      auto-resets <form action={fn}> after submission — even on a
      fieldErrors return — wiping uncontrolled input values. Fix
      pattern: action echoes submitted values back in state.values;
      form re-renders via a formGen key bump so the echoed values
      become the new defaultValue at remount. (b) z.enum().optional()
      .nullable() rejects "" (empty string) but the <select> with an
      empty placeholder option always sends "". Fix pattern: wrap the
      enum in a preprocess that maps "" → null first.
    - SELECT ERROR HIGHLIGHTING was missing on tenant forms — the
      Field component supported it but the Select component didn't.
      Added error prop + red-border treatment + per-key error wiring
      in CustomerFormFields + InventoryFormFields. The "fix the
      highlighted fields" banner without a highlighted field was the
      symptom that exposed both bugs.
    - STRIPE REST PATTERN LOCKED. Every Stripe call (Connect from
      Phase 5, SaaS from Phase 9 A) uses raw fetch + URLSearchParams
      against api.stripe.com — no SDK dependency, Edge-safe, identical
      idempotency-key behavior. Webhook signature verify uses Web
      Crypto HMAC-SHA256 with 5-min tolerance.
    - PLAN-TIER FEATURE GATES resolution: tenant_subscriptions.plan_id
      wins; falls back to lowest-priced active plan (sort_order ASC).
      Status is intentionally NOT considered — past_due / unpaid
      degradation is dunning's job (chunk 5), not the gate's, so a
      temporary card decline doesn't shut off paid features mid-shift.
    - SAAS DUNNING uses platform Resend (RESEND_API_KEY env), NOT
      per-tenant creds. message_log row attaches to the recipient
      tenant; customer_id=null since the recipient is a staff user
      (the tenant owner), not a customer record. Patches/0016 added
      4 saas_* MessageKinds for this.
    - BUY-OUTRIGHT compliance pattern: ONE compliance_log row per
      buy transaction, items_snapshot lists every item, source_table
      ='inventory_items', source_id=first inventory item's id.
      Mirrors pawn_intake's items_snapshot shape; FL LeadsOnline
      exporter already flattens 1 row → N police-report rows.
    - 1099-MISC reads EXCLUSIVELY from compliance_log (Rule 15 — the
      same table the police report reads from). Single source of
      truth means the IRS picture and the police picture can never
      drift. v1 has no SSN/TIN field; CSV emits a labeled
      "(collect via W-9)" placeholder so the accountant has a slot.
    - PORTAL STRIPE-OPTIONAL: render path is now fully decoupled from
      Stripe Connect status. isTenantStripeConnected() is a non-
      throwing companion to getStripeAccountIdForTenant(). The "Pay
      online" button hides + a "pay in store" notice replaces it
      when Connect isn't onboarded. Customers can see balances + due
      dates either way.
    - 24K GOLD HEADLINE on /inventory/spot-prices is the operator's
      morning anchor price. Big tabular numerals at the top of the
      page; always renders when the spot row exists, regardless of
      the per-card hide list.
    - PER-CARD HIDE persists in localStorage under pawn.spot-prices.
      hidden — same per-device pattern as the sidebar collapse.
    - PARALLEL-AGENT WORKTREE pattern continues to work but has a
      gotcha — agents that write before realizing they're in a
      worktree can leak files to the main filesystem. The 1099 agent
      hit this and recovered via `git show main:<path> > <main-path>`
      to restore + rm to clean. Lesson: agents should `pwd` before
      first Write to confirm they're in the worktree path.
