      (tenant_id, license_key) — same shape) and is a one-line
      directive instead of a wider refactor.
    - NEXT_PUBLIC_APP_URL IS THE CANONICAL ORIGIN SOURCE. Don't
      `typeof window !== 'undefined'` to switch between absolute
      and relative URLs in shared components — it causes hydration
      mismatch. Use the env var; it's inlined at build time and
      identical on both render passes.
    - WINDOW.LOCATION.ASSIGN OVER ROUTER.PUSH FOR COOKIE-SET
      NAVIGATIONS. router.push doesn't always re-run middleware
      against the new cookie reliably; window.location forces a
      fresh request that the proxy + getCtx see with the updated
      cookie. Used in TenantSwitcher's onSwitch and /admin/tenants
      Open buttons.
    - "LOOKUP BY EMAIL" SQL PATTERN for inserting profile rows
      tied to auth.users. Replaces "copy the UUID manually" with
        INSERT INTO profiles (id, role, ...) SELECT id, 'superadmin', ...
        FROM auth.users WHERE email = '<email>'
        ON CONFLICT (id) DO UPDATE SET ...
      Idempotent, lookup-by-email, no manual UUID step. Save as a
      /convention candidate next session.
