  KEY DECISIONS LOCKED THIS SESSION:
    - PARALLEL-AGENT WORKTREE PATTERN. Three agents ran in worktree
      isolation but, by happy accident, all also wrote to the primary
      tree because each read the in-flight overlay before writing.
      Final commit went straight from primary tree, not from the
      worktree branches. Worktrees are now treated as scratch /
      isolation only; final integration runs from main. Don't
      formalize this — note it as observed.
    - COMPLIANCE_LOG WRITTEN AT INTAKE, NOT DERIVED. The pawn intake
      action emits a compliance_log row inside the same transaction
      as the loans + loan_collateral_items insert. No retroactive
      derivation at police-report time. Locks the data the way the
      regulator expects.
    - FORFEIT-TO-INVENTORY HOOK SEMANTICS. forfeitLoanAction copies
      every loan_collateral_items row into inventory_items with
      source='pawn_forfeit' and source_loan_id=<loan.id>. Loan status
      flips to 'forfeited'. Item status starts 'available' (not
      'held') because forfeiture already cleared the customer's claim
      — the hold-period rule applies to BUY-OUTRIGHT, not pawn
      forfeitures.
    - PAWN TICKET PRINT IMMUTABILITY WITH EXTENDED-STATUS CARVE-OUT.
      Database trigger locks principal / interest_rate / term_days /
      due_date / customer_id / collateral once is_printed=true,
      EXCEPT when status transitions to 'extended' (extension is the
      one legitimate post-print mutation — it advances due_date +
      writes a loan_events row).
    - DECIMAL-SAFE LOAN MATH IN PURE FUNCTIONS. lib/pawn/math.ts is
      string-/Number-based with explicit r4 rounding to 4 decimal
      places. interestAccrued is straight-line per-day; payoffBalance
      = principal + accrued interest − interest already paid (floored
      at 0); splitPayment is interest-first (interest portion satisfied
      before principal). NO floats anywhere.
    - AUDIT-FORMAT HELPER SEPARATED FROM AUDIT.TS. lib/audit.ts
      imports the service-role admin client; if a client component
      pulled in label-formatting from there, the admin client would
      land in the client bundle. Moved the formatter to
      lib/audit-format.ts (pure helper, no Supabase import) so the
      /audit ChangesViewer can stay client-side without dragging
      service-role code along.
    - PER-TABLE AUDIT LABEL OVERRIDES. The same action verb ('create',
      'update', 'soft_delete') means different things per table — i18n
      provides audit.actions.perTable[table][action] as an override
      map; falls back to audit.actions[action], then to the raw action
      string. Display layer only — write side stays canonical.
    - 'USE SERVER' EXPORTS RULE. Next 16 / React 19 enforce that a
      'use server' module can only export async functions. A const
      object trips a runtime crash. Type exports are fine (erased at
      compile). Pattern going forward: never export const ...InitialState
      — inline `{}` at the useActionState call site, keep the State
      type export for typing. Documented in fix commit a481a82.
    - INVENTORY EDIT/DELETE LOCKED ON status='transferred'. While an
      item is mid-transfer it can't be edited or soft-deleted. Action
      returns 'item_locked_in_transfer' field error. Sits alongside
      the existing block on status='sold' / 'scrapped'.
    - SCRIPTS/APPLY-MIGRATION.MJS. Wraps the Supabase Management API
      `POST /v1/projects/{ref}/database/query` so SQL migrations can
      be applied from CLI without copy-pasting into the SQL Editor.
      Reads SUPABASE_PROJECT_ID + SUPABASE_ACCESS_TOKEN from
      .env.local. Used for 0005 + 0006 in this session.
